### v3.67.0 — LazyClaw dashboard + provider/chat/term parity wave (Ralph cycle)

**Branch**: `main` (interactive — Ralph loop user request)

**User goal (Korean)**: 워크플로우 세션 에러/최적화 전체 해결, lazyclaw 렉 알고리즘 기반 최적화, 하드코딩·더미데이터 제거, 프로바이더 전체 최적화, 기본모델 등 미구현 항목·error 표기 구현, lazyclaw 대시보드, AI 채팅에서 하위 에이전트 호출, 터미널 openclaw 동등 구현. Playwright 검증 필수.

**Findings**:
- `saveDefaultModel` was a no-op — posted `defaultModel` to
  `/api/ai-providers/save-key` which only reads `apiKey` / `baseUrl`.
  The proper endpoint `/api/ai-providers/default-model` already
  existed but was never wired up. Backend `/list` also wasn't
  stamping `isDefault` on each model dict, so even if the save had
  worked the UI's preselect path would have read undefined.
- Chat streaming was rebuilding ALL messages via
  `innerHTML = history.map(...).join('')` every 30 ms. With marked.parse
  per message, long sessions stutter visibly.
- 4 frontend call sites bypassed the existing
  `cachedApi('/api/ai-providers/list')` 30 s TTL and re-fetched on
  every render of orchestrator / lazyclawChat / lazyclawTerm /
  workflow run-error modal.
- LazyClaw mode had no dashboard — its default tab was
  `orchestrator`, which only covers Slack/Telegram dispatch. Nothing
  surfaced provider state, chat sessions, or workflow telemetry as a
  hub.
- `lazyclawTerm` lacked `agents | sessions | skills | doctor` verbs
  that exist in the lazyclaw CLI.

**Changes**:
- `dist/app.js`:
  - New `VIEWS.lazyclawDashboard` + tile-based hub (provider grid,
    workflow stats, recent chat sessions, orchestrator history).
  - `_lcPatchStreamMsg(idx, msg)` patches only the streaming bubble
    via `data-lc-msg-idx` instead of full re-render. Streaming token
    throttle 30 ms → 60 ms.
  - 4× non-cached `/api/ai-providers/list` swapped to `cachedApi`.
  - `saveDefaultModel` rewritten to hit `/api/ai-providers/default-model`
    with `_apiCacheInvalidate` after success.
  - `/delegate` and `/parallel` chat slash commands.
  - `lazycodex agents | sessions | skills | doctor` terminal verbs.
  - All command lists (KNOWN_VERBS, candidates, _ALL suggestions,
    Tab cycler `cmds`, help groups) updated in lockstep — single
    edit-set required when adding a verb.
- `server/ai_keys.py`: `api_providers_list` post-processes the
  parallel `_probe()` output — stamps each provider with
  `defaultModel: defaults[p.id]` and tags the matching `models[]`
  entry with `isDefault: True`.
- `server/nav_catalog.py`: `lazyclawDashboard` tab metadata.
- `scripts/e2e-v3-67-verify.mjs`: targeted Playwright suite.

**Verification (Playwright)**:
- `e2e-tabs-smoke` — 67/67 tabs (was 66 before catalog entry).
- `e2e-chat-slash-smoke` — every existing slash still handled.
